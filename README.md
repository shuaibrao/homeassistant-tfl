# TfL Tube Status Card

A premium, authentic custom dashboard card for Home Assistant that replicates the live status widget found on the Transport for London (TfL) website. 

It highlights lines with active disruptions (such as minor delays, severe delays, or closures) individually at the top and collapses all operational lines into a sleek, compact stack of color stripes at the bottom.

## Features

- 🚇 **Authentic TfL Styling**: Replicates the font spacing, borders, cream status backgrounds (`#FAF6E9`), corporate blue text (`#00205B`), and official line colors.
- ⚡ **Zero-Configuration Default**: Instantly fetches live status directly from the TfL API. No API keys or integrations required.
- 🔄 **Smart Request Caching**: Implements a global cache wrapper in the browser so that multiple cards across your dashboard do not spam the TfL API.
- 🔌 **Home Assistant Entity Integration**: Offline-friendly! Can read line status directly from the entities generated by Home Assistant's official London Underground integration.
- 📱 **Interactive Accordion Details**: Delayed lines smoothly expand when clicked to show detailed disruption reasons. Clicking the "Good service on all other lines" row lists all running lines in a clean grid.
- 🎨 **Named Overground Support**: Fully supports the six new named London Overground lines (Liberty, Lioness, Mildmay, Suffragette, Weaver, Windrush) with their correct colors.
- 🔧 **Highly Configurable**: Easily filter by specific transport modes, filter specific lines, toggle between the grouped and detailed views, and adjust polling intervals.

## Installation

### Method 1: HACS (Recommended)

1. Open **HACS** (Home Assistant Community Store) in your Home Assistant panel.
2. Click the three dots `...` in the top right corner and select **Custom repositories**.
3. Paste the URL of this repository: `https://github.com/shuaibrao/homeassistant-tfl`.
4. Set the Type to **Dashboard**.
5. Click **Add**.
6. Find "TfL Tube Status Card" in HACS, click **Download**, and restart your frontend if prompted (HACS will register the resource automatically).

### Method 2: Manual Installation

1. Download the `ha-tfl-card.js` file.
2. Upload it to your Home Assistant configuration directory under `www/` (e.g., `/config/www/ha-tfl-card.js`).
3. Add the Lovelace resource in Home Assistant:
   - Go to **Settings** > **Dashboards** > Click the three dots `...` (top right) > **Resources**.
   - Click **Add Resource**.
   - Enter `/local/ha-tfl-card.js` as the URL (Note: `/local/` corresponds to the `www/` folder).
   - Set the Resource Type to **JavaScript Module**.
   - Click **Create**.

## Configuration

Add a manual card to your Lovelace dashboard and enter your configuration:

### 1. Direct Mode (Default, Zero-Config)
No configuration required. It will fetch all lines (Tube, DLR, Elizabeth line, Overground named lines, Tram) directly from the TfL API.

```yaml
type: custom:ha-tfl-card
```

### 2. Custom Title & Line Filtering
Only monitor specific lines (e.g., Victoria and Elizabeth line) and change the card title.

```yaml
type: custom:ha-tfl-card
title: "My Commute Status"
show_lines:
  - victoria
  - elizabeth
```

### 3. Mode Filtering
Only show Tube lines and the DLR, ignoring Trams and Overground.

```yaml
type: custom:ha-tfl-card
show_modes:
  - tube
  - dlr
```

### 4. Detailed View (No Grouping)
If you prefer listing every line individually (disabling the grouped stripes view):

```yaml
type: custom:ha-tfl-card
show_all_lines: true
```

### 5. Entity Mode (Offline-friendly using HA Integration)
If you already use Home Assistant's built-in **London Underground** integration, you can map the card to read directly from those entities:

```yaml
type: custom:ha-tfl-card
title: "TfL Integration Status"
entities:
  - sensor.london_underground_bakerloo
  - sensor.london_underground_central
  - sensor.london_underground_district
  - sensor.london_underground_jubilee
  - sensor.london_underground_northern
  - sensor.london_underground_victoria
```
*Note: The card automatically maps the entity IDs to their respective TfL line colors and metadata.*

## Configuration Schema

| Option | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| **`type`** | string | **Required** | Must be `custom:ha-tfl-card` |
| **`title`** | string | `Tube, DLR...` | Custom header title |
| **`show_all_lines`** | boolean | `false` | Set `true` to list all lines individually instead of grouping good service lines. |
| **`update_interval`** | number | `60` | Polling frequency in seconds (applies to Direct API fetching only). |
| **`api_key`** | string | `null` | Optional TfL Unified API key to increase request rate limits. |
| **`show_modes`** | list | `null` | Transport modes to show: `tube`, `dlr`, `overground`, `elizabeth-line`, `tram`. |
| **`show_lines`** | list | `null` | Specific line IDs to show (e.g. `central`, `victoria`, `windrush`). |
| **`entities`** | list | `null` | Custom Home Assistant entities. Switch card to offline/entity-driven mode. |

## Development

If you'd like to test the card locally without placing it inside a live Home Assistant environment, you can launch a local dashboard simulator:

1. Create a dummy dashboard HTML page.
2. Load the card using a `<script type="module" src="./ha-tfl-card.js"></script>`.
3. Add the card element `<ha-tfl-card></ha-tfl-card>` with mock configurations.
